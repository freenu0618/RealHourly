# RealHourly Demo Script (2min)

> "Your contract says $75/hr. Your bank account says $23. Find your real rate."

---

## 1. 데모 전 준비

### 1.1 환경 체크리스트

| # | 항목 | 확인 |
|---|------|------|
| 1 | `.env.local` 설정 완료 (DATABASE_URL, OPENAI_API_KEY, SUPABASE_*, LLM_MODEL_*) | [ ] |
| 2 | `pnpm dev` 실행 중 (또는 Vercel 배포 URL 접속 가능) | [ ] |
| 3 | Supabase 유저 1명 이상 존재 + 로그인 상태 | [ ] |
| 4 | `pnpm seed` 실행 완료 (시드 데이터 삽입) | [ ] |
| 5 | `pnpm verify-seed` 실행 → 38/38 통과 | [ ] |
| 6 | 브라우저 언어: 한국어 (자동 `/ko` 라우팅 확인) | [ ] |

### 1.2 브라우저 탭 (미리 열기, 좌→우 순서)

| 탭 # | URL | 용도 |
|-------|-----|------|
| 1 | `/ko/projects` | 장면 1: 프로젝트 목록 (첫 화면) |
| 2 | `/ko/projects/{ABC리브랜딩ID}` | 장면 2-3: 팩트 폭격 + 스코프 경고 |
| 3 | `/ko/time-log` | 장면 4: 자연어 타임로그 |

> **Tip**: 탭 2는 열면 자동으로 모달이 뜨므로, 데모 직전에는 탭 1만 활성화 상태로 둘 것.
> 탭 2를 미리 로드하면 모달이 이미 뜨거나 dismiss될 수 있음. **탭 2는 탭 주소만 입력해두고 Enter 치지 말 것** — 장면 2 시작 시 직접 이동.

### 1.3 화면 설정

| 항목 | 권장값 |
|------|--------|
| 해상도 | 1920x1080 (16:9) 또는 1280x720 |
| 브라우저 확대 | 100% (발표장 스크린이 멀면 110~125%) |
| 개발자 도구 | **닫기** |
| 북마크 바 | 숨기기 (F11 전체화면 or Cmd+Shift+B) |
| 다크 모드 | 취향 (둘 다 지원됨, 다크모드가 차트 대비 좋음) |
| 알림/팝업 | 모두 끄기 (방해 금지 모드) |

### 1.4 리허설 체크

- [ ] "예시 1 채우기" 버튼 → Magic Parse → 카드 나타남 확인 (OpenAI 응답 속도 테스트)
- [ ] ABC 리브랜딩 상세 → 모달 뜨는지 확인
- [ ] "메시지 생성" → 3개 탭 나타나는지 확인 (OpenAI 응답 속도 테스트)
- [ ] 복사 버튼 → 클립보드 작동 확인

## 4. 백업 플랜

### 4.1 LLM 파싱 실패 시 (장면 4)

| 증상 | 대처 |
|------|------|
| Magic Parse 로딩이 10초 이상 | "AI 응답이 좀 느리네요. 다른 방법도 있습니다" 하며 수동 입력 폼 전환 |
| 500 에러 / 파싱 실패 토스트 | "예시 채우기" 버튼으로 재시도. 같은 에러면 수동 입력 폼으로 전환 |
| 프로젝트 매칭 실패 (PROJECT_UNMATCHED) | 드래프트 카드에서 프로젝트 드롭다운 직접 선택 → "이렇게 수동 보정도 가능합니다" |

**대사 (실패 시)**:
> "네트워크 상태에 따라 AI 응답이 늦을 수 있는데, 이럴 때는 수동 입력도 가능합니다. 핵심은 분 단위만 입력하면 된다는 점이죠."

### 4.2 메시지 생성 실패 시 (장면 3B)

| 증상 | 대처 |
|------|------|
| 로딩 10초 이상 | "잠시만요..." 기다리기 (fallback 메시지가 자동 생성됨) |
| 에러 토스트 | LLM 실패 시 자동으로 template-based fallback 메시지 반환됨 (코드상 자동) |
| Fallback도 실패 | "네트워크 이슈네요" → 모달 닫고 다른 장면으로 넘기기 |

> **참고**: `generate-messages.ts`에 fallback 로직이 내장되어 있어, OpenAI API 실패 시 한국어/영어 템플릿 메시지가 자동 반환됩니다. 실제로 빈 화면이 될 확률은 매우 낮습니다.

### 4.3 네트워크 느릴 때

| 상황 | 대처 |
|------|------|
| Vercel 배포 느림 | `localhost:3000`으로 전환 (미리 `pnpm dev` 실행) |
| WiFi 불안정 | 핫스팟 준비 또는 유선 연결 |
| API 전체 다운 | 시드 데이터 기반으로 프로젝트 목록 + 상세는 보여줄 수 있음 (SSR). 타임로그/메시지만 건너뛰기 |

### 4.4 모달이 안 뜰 때 (장면 3)

| 원인 | 대처 |
|------|------|
| 이전 리허설에서 Dismiss 했음 | `pnpm seed` 재실행 (idempotent — 기존 데이터 삭제 후 재삽입) |
| alert가 없음 | 프로젝트 상세 페이지 새로고침 → `getProjectMetrics`가 alert를 자동 생성 |

### 4.5 사전 안전 장치

```bash
# 데모 직전 실행 (시드 데이터 초기화)
pnpm seed

# 시드 검증
pnpm verify-seed
```

## 5. 예상 Q&A

### 5.1 제품/비즈니스 질문

| 질문 | 답변 |
|------|------|
| **Toggl, Harvest와 뭐가 달라요?** | Toggl은 시간 추적만 합니다. RealHourly는 숨은 비용(수수료, 세금, 비청구 시간)까지 자동 반영해서 **진짜 시급**을 계산하고, 스코프 크립을 감지해서 **추가 청구 메시지까지** 생성해줍니다. 시간 추적 → 수익성 분석 → 행동 유도까지 한 사이클입니다. |
| **ChatGPT로 직접 하면 안 되나요?** | ChatGPT는 개인 데이터 누적이 불가하고, 실시간 스코프 크립 감지도 못합니다. "이번 달 revision이 40% 초과"를 감지하려면 프로젝트별 time_entries가 DB에 있어야 합니다. 또한 Structured Outputs로 파싱 정확도가 일반 프롬프트보다 높습니다. |
| **수익 모델은?** | 월 $19~$39 구독. 무료 플랜(1프로젝트) + Pro 플랜(무제한 프로젝트 + AI 메시지). 유료 전환 500명 시 $10K MRR 예상. |
| **시장 규모?** | 전 세계 자영/독립 노동자 15억+ 중 플랫폼 활성 이용자 수천만 명. 프리랜서 플랫폼 시장 연 15% 성장. |
| **경쟁 우위?** | ROI가 눈에 보임. "이 도구를 써서 $500 추가 청구했다" — 구독비 $19로 $500 벌었으면 25배 ROI. |
| **해외 진출은?** | i18n 기반 구축 완료 (한/영). 5개 통화 지원 (USD, KRW, EUR, GBP, JPY). 브라우저 언어 자동 감지. |

### 5.2 기술 질문

| 질문 | 답변 |
|------|------|
| **스택?** | Next.js 16 (App Router) + TypeScript + Tailwind + shadcn/ui. DB는 Supabase (PostgreSQL + Auth + RLS). ORM은 Drizzle, LLM은 OpenAI gpt-5-mini. |
| **LLM 비용은?** | gpt-5-mini 기준 파싱 1회 ~$0.002, 메시지 생성 1회 ~$0.005. 월 1,000명 기준 ~$50/월. |
| **파싱 정확도?** | OpenAI Structured Outputs (strict JSON Schema) 사용. LLM은 구조화만 담당하고, 프로젝트 매칭/날짜 계산/유효성 검증은 서버에서 수행. LLM이 틀려도 HITL(Human-In-The-Loop) 카드에서 사용자가 보정. |
| **확장성?** | Supabase PostgreSQL (수직 확장). Vercel Edge Functions (수평 확장). DB 쿼리는 인덱스 최적화 완료. LLM은 OpenAI API라 인프라 관리 불필요. |
| **보안?** | Supabase RLS (Row Level Security)로 모든 테이블 유저 격리. API 진입점에서 `requireUser()` 인증 필수. 소프트 삭제로 데이터 복구 가능. |
| **스코프 크립 어떻게 감지?** | 3가지 규칙 기반: (1) 예상 시간 80% 소진 + 진척 50% 미만, (2) 수정 작업이 전체의 40% 이상, (3) 수정 이벤트 5회 이상. LLM 아닌 deterministic 규칙이라 예측 가능하고 설명 가능. |
| **왜 Server Actions 안 쓰나요?** | P0 MVP 단계에서 디버깅 용이성 우선. Route Handlers는 cURL/Postman으로 직접 테스트 가능하고, 에러 추적이 명확합니다. P1에서 Server Actions 전환 검토 중. |

## 7. 발표자 노트

### 톤 & 에너지
- **시작**: 차분하게 문제 제기 ("프리랜서 여러분...")
- **팩트 폭격**: 임팩트 있게 ("$50이 $28.72가 됩니다!")
- **스코프 경고**: 공감 유발 ("이런 상황 겪어보셨죠?")
- **메시지 생성**: 해결 제시 ("AI가 대신 써줍니다")
- **마무리**: 자신감 있게 ("프리랜서의 돈을 지켜줍니다")

### 흔한 실수 방지
- ❌ 코드나 아키텍처 설명하지 않기 (Q&A에서 답변)
- ❌ "P0 MVP라서..." 변명하지 않기
- ❌ 로딩 시간에 침묵하지 않기 (설명으로 채우기)
- ✅ 숫자를 강조하기 ("$50 → $28" 반복)
- ✅ 문제 → 해결 흐름 유지하기
- ✅ 실제 데이터로 신뢰감 주기
